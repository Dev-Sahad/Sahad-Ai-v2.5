# 🏛️ Project Architecture

This project is a standalone Node.js application that bridges the gap between low-latency AI responses and high-security user sessions. It is designed to run efficiently on cloud platforms like **Render** while maintaining a lightweight footprint.

---

# 🛡️ Security & Authentication

- **JWT (JSON Web Tokens):**  
  Implements a stateless authentication system. Once authenticated, the browser stores a signed token valid for 7 days.

- **Environment Isolation:**  
  Sensitive keys (`API secrets`, `passwords`) are managed via `process.env` to prevent exposure.

---

# 👁️ Multimodal Capabilities (Vision)

The core engine is configured to process `base64` image data, allowing the AI to "see" your work.

### Supported Use Cases

# ⚙️ Configuration (Environment Variables)

To deploy successfully, add the following variables to your Render dashboard or local `.env` file.

| Variable | Description |
| :--- | :--- |
| `API_KEY` | Official Google Gemini API Key |
| `JWT_SECRET` | Random secure string for signing tokens |
| `MY_PASSWORD` | Password used to unlock the AI workspace |
| `PORT` | Application port (`3000` recommended) |

# 🚀 Deployment Guide

## 1️⃣ Cloud Deployment (Render)

### Step 1 — Connect Repository
Create a new **Web Service** on Render and connect your GitHub repository.

### Step 2 — Add Environment Variables
Navigate to:
```bash
Render Dashboard → Environment → Add Secret Keys
```

Paste all required `.env` variables.

### Step 3 — Configure Build Settings

#### Build Command
```bash
npm install
```

#### Start Command
```bash
node server.js
```

### Step 4 — Enable Auto Deploy
Every push to GitHub automatically redeploys the application.
